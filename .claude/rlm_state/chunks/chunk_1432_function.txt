# Common Pitfalls to Avoid

# Introduction to Decorators

Decorators are a powerful and expressive feature in Python that allows you to modify or enhance functions and classes in a clean and reusable way. They provide a way to wrap additional functionality around an existing function or class without modifying its core logic. This promotes code reusability, improves readability, and simplifies maintenance.

In essence, a decorator is a function that takes another function as an argument, adds some functionality, and returns a new function. This new function is often referred to as a "wrapped" function.

## Why Use Decorators?

*   **Code Reusability:** Decorators allow you to apply the same modification to multiple functions or classes without repeating the code.
*   **Readability:** They make your code more readable by separating the core logic from the additional functionality.
*   **Maintainability:** Changes to the added functionality only need to be made in one place, making maintenance easier.
*   **DRY (Don't Repeat Yourself) principle:** Decorators are a perfect example of the DRY principle.

# Function Decorators

Function decorators are the most common type of decorator. They are used to modify or enhance the behavior of a function.
